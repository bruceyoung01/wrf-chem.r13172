Overview

pop_to_dart is the program that reads a POP restart file (usually pop.r.nc) and creates a DART output/restart file (e.g. perfect_ics, filter_ics, ... ).

The following variables are extracted from the POP netCDF restart files and are conveyed to DART: SALT_CUR, TEMP_CUR, UVEL_CUR, VVEL_CUR, and PSURF_CUR.

Conditions required for successful execution of pop_to_dart:

  • a valid input.nml namelist file for DART
  • a valid pop_in namelist file for POP
  • the POP geometry files mentioned in the pop_in namelist file
  • a POP restart file (pop.r.nc).

Since this program is called repeatedly for every ensemble member, we have found it convenient to link the POP restart files to the default input filename (pop.r.nc). The default DART output filename is dart_ics - this may be moved or linked as necessary.

NAMELIST

This namelist is read from the file input.nml. Namelists start with an ampersand '&' and terminate with a slash '/'. Character strings that contain a '/' must be enclosed in quotes to prevent them from prematurely terminating the namelist. 

&pop_to_dart_nml
   pop_to_dart_output_file = 'dart_ics'  
/


Item Type Description
pop_to_dart_output_file character(len=128) The name of the DART file containing the model state derived from the POP restart file.
